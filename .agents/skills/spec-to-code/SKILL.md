---
name: spec-to-code
description: Convert a specification into an executable implementation plan and delivery sequence.
---

1) Read the spec and extract requirements, constraints, and non-goals.
2) Map impacted components and interfaces in the repository.
3) Produce phased tasks with acceptance criteria and file targets.
4) Define deterministic tests for each phase.
5) Output:
   - assumptions
   - phased task list
   - verification commands
   - risks and rollback notes
