---
name: repo-onboarding
description: Build a quick architecture and workflow map for safe contribution.
---

1) Identify repo layout, entry points, and dependency boundaries.
2) Locate install/build/lint/typecheck/test commands.
3) Identify sensitive paths and security-critical modules.
4) Summarize coding conventions and test expectations.
5) Output:
   - repo map
   - command checklist
   - risk areas
   - first-task recommendations
